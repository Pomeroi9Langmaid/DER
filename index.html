DER asked

Data coverage: which destinations are available and how granular the data is.

Smartvel response

The Destination Coverage file provides available destinations, filterable by regional area, country and destination. Smartvel coverage is structured at destination-city level, with POI-level content including name, description, image/image reference, geodata and tags.

DER asked

Provide hotel geodata and receive content assigned to that location or nearby area.

Smartvel response

Yes, this is conceptually aligned with Smartvel’s POI API. The hotel location can act as the anchor point for nearby POI and destination content. The exact logic depends on radius, coverage, content types, languages and matching rules.

Commercial scoping

What we need from DER to provide a rough cost indication

To provide a sensible commercial indication for the POI data/API requirement, Smartvel would first need to confirm:

A. Use case

  • API-only data access?
  • API + widget support?
  • Both options?

B. Brand / app

  • Which brand or app environment?
  • Connected to the DER app requirement?

C. Destinations

  • Which destination cities / markets?
  • Crete, Mallorca and Turkey: same scope?
  • Which Turkey destination?

D. Languages

  • Which languages are required?

E. Hotel geodata logic

  • Radius-based?
  • Destination-city based?
  • Another geographic rule?

F. Content types

  • POI/editorial content only?
  • Tours / activities / events also?

G. Provider feeds

  • External tour/activity providers?
  • Any provider feeds or APIs?

H. Tags and filters

  • Operational filters?
  • Editorial/profile tags?
  • Accessibility requirements?
